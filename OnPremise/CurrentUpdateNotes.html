Update Notes for Splice Machine Version 2.5

This topic describes significant updates in this release and recent, previous releases of Splice Machine.

  • General Notes includes notes that apply to all Splice Machine releases.
  • Updates in This Version includes notes about new updates in this release.

Updates in This Version

This section summarizes significant updates to the Splice Machine software and documentation since the previous release, which was version 2.0. This section includes the following sections:

  • Performance Enhancements
  • New and Updated Features
  • Deprecated Features
  • Documentation Updates

Performance Enhancements

This release features a number of significant performance improvements:

  • Performance enhancements on TPCC and TPCH benchmarks.
  • Greatly improved data importation performance.
  • The ability to tell Splice Machine to pin (cache) a table in memory to allow rapid access for analytical tasks.
  • Performance enhancements to our JDBC and ODBC drivers.
  • Several of the window functions have been revised for better performance.

New and Updated Features

This section summarizes important feature updates that have been made to the current release (Product Release Version 2.5) of Splice Machine since release 2.0.

Change Description
Backup and Restore

The ability to incrementally back up and restore your database is now available to Splice Machine Enterprise Edition users.

For more information, see Backing Up and Restoring Your Database in our Administrator's Guide, and the following built-in system procedures and system tables in our SQL Reference Manual:

  • SYSCS_UTIL.SYSCS_BACKUP_DATABASE
  • SYSCS_UTIL.SYSCS_CANCEL_BACKUP
  • SYSCS_UTIL.SYSCS_CANCEL_DAILY_BACKUP
  • SYSCS_UTIL.SYSCS_DELETE_BACKUP
  • SYSCS_UTIL.SYSCS_DELETE_OLD_BACKUPS
  • SYSCS_UTIL.SYSCS_RESTORE_DATABASE
  • SYSCS_UTIL.SYSCS_SCHEDULE_DAILY_BACKUP
  • SYSBACKUP System Table
  • SYSBACKUPITEMS System Table
  • SYSBACKUPJOBS System Table
External Tables

You can now access ORC and PARQUET files as external tables. See Using External Tables in our Developer's Guide for more information.

The CREATE EXTERNAL TABLE statement and the SYSCS_UTIL.SYSCS_REFRESH_EXTERNAL_TABLE built-in system procedure are now available for working with external tables.
Migration Assistant Tool

The Migration Assistant tool was created by Splice Machine to help migrate your database to a new version of the Splice Machine software. You run it before upgrading your software to export your database to file, and then run it after upgrading your software to import an upgraded version of your database.
Pinned Tables
(RDD Caching)

You can now use the PIN TABLE statement to cache a static version of a table in memory, and use the UNPIN TABLE statement to delete the cached version of the table.

You also need to use the new pin query optimization hint, which is described in our Query Optimization topic.
Schema-level Privileges Updates to the Grant and Revoke statements allow you to define privileges at the schema level.
Updated Connectors

Splice Machine has added enhancements to our JDBC and ODBC drivers for improved performance and added features.

Due to these changes, you must use the Splice Machine JDBC and ODBC drivers; other drivers will not work properly.
Updated Statistics

Splice Machine has enhanced our statistics collection methods, taking advantage of advanced features provided by the Yahoo DataSketches library. As a result, these procedures and tables have been deprecated:

  • SYSCS_UTIL.SYSCS_COLLECT_TABLE_STATISTICS
  • SYSCS_UTIL.SYSCS_DROP_TABLE_STATISTICS
  • SYSSTATISTICS table

Deprecated Features

Change Description

The following built-in system procedures are no longer available:

  • SYSCS_UTIL.SYSCS_COLLECT_TABLE_STATISTICS
  • SYSCS_UTIL.SYSCS_DROP_TABLE_STATISTICS

Note that:

  • You can continue to use the ANALYZE command line command to collect statistics for a specific table.
  • You can call the SYSCS_UTIL.SYSCS_COLLECT_SCHEMA_STATISTICS built-in system procedure to collect statistics on a schema, including statistics for every table in the schema.
Statistics Tables

The following system tables are no longer available:

  • SYSSTATISTICS
  • SYSOPERATIONHISTORY
  • SYSSTATEMENTHISTORY
  • SYSSTATISTICS
  • SYSTASKHISTORY

New and Updated Features

The following table summarizes important feature updates that have been completed in recent releases.

Version Updated Change Description
2.0 Added functions The IDENTITY_VAL_LOCAL function is now available in Splice Machine.
2.0 Common Table Expressions Common table expressions, also known as the WITH clause or the subquery factoring clause, are now available in Splice Machine.
2.0 Data Import changes

Our data import technology was modified for improved performance and error handling. The number of parameters and the meaning of certain parameters changed for these built-in system procedures:

  • SYSCS_UTIL.IMPORT_DATA
  • SYSCS_UTIL.UPSERT_DATA_FROM_FILE
2.0 Timestamps in imported data

Previous versions of Splice Machine were "lenient" with regard to parsing timestamps when importing data:

  • Timestamp data in imported columns must now match the formatting pattern you specify in the IMPORT or UPSERT call.
  • All timestamp values in a file being imported must now use the same timestamp format; multiple timestamp formats in a file are no longer supported.
  • Splice Machine now uses only one default pattern: yyyy-MM-dd HH:mm:ss, which specifies a 24-hour clock and no fractions of seconds.
2.0 ODBC Driver The new and improved Using the Splice Machine ODBC Driver driver is now available.
2.0 Schema owner

The owner of a schema can now be changed by the current schema owner:

    SYSCS_UTIL.SYSCS_UPDATE_SCHEMA_OWNER
2.0 Spark engine

As described in the Performance Enhancements section above, we now use Spark for greatly improved query execution performance for analytical queries.
2.0 Temporary Tables The previous restrictions on data types allowed in temporary tables have been removed: you can now use all data types in temporary tables.
1.5.5 Cloudera release compatibility Splice Machine now works with Cloudera 5.4.10, in addition to earlier versions of Cloudera, as noted in our Splice Machine Requirements page.
1.54 Import Logging

When you enable import logging by setting the property splice.import.enableStatusLogging to TRUE, you can view the logged information in two ways:

  • In the region server logs
  • In the summary file (this is new) that is created in the /BAD directory that you specified in your call to SYSCS_UTIL.IMPORT_DATA call.
1.53 Cloudera release compatibility

Splice Machine now works with Cloudera 5.4.8, in addition to earlier versions of Cloudera, as noted in our Splice Machine Requirementspage.
1.51 Synonyms

The following SQL statements are now available:

  • CREATE SYNONYM
  • DROP SYNONYM
1.50 Auto Trace

The auto trace feature was previously on by default; it has been changed to be off by default. You can turn it on using the Auto Trace built-in system procedure.
1.50 Date Arithmetic

A number of new date functions have been added to enhance our compatibility with third party Business Intelligence tools. There's A new topic in the Developer's Guide: Working With Dates.

We have added these new built-in SQL date and date arithmetic functions:

  • EXTRACT
  • MONTHNAME
  • NOW
  • QUARTER
  • ROWID
  • WEEK

The SECOND function has been updated to work properly with timestamp values.

And there's a significantly updated version of the TRUNC or TRUNCATE function that now works with date values. As a result, the TRUNC_DATE function has been deprecated.
1.50 DBeaver SQL Client We have added instructions for installing and Using DBeaver with Splice Machine.
1.50 Foreign Keys

Foreign keys are now operational. There's a new topic in the Developer's Guide: Foreign Keys and Referential Integrity.

And there are updates for foreign keys in these SQL statements:

  • ALTER TABLE
  • CONSTRAINT
  • CREATE TABLE
1.50 Incremental backup

Both full and incremental backups are now supported. There's a new topic Backing Up and Restoring Databases topic in the Maintaining section of this Guide.

We've made major updates to these functions, including the ability to backup to and restore from cloud services such as AWS:

  • SYSCS_UTIL.SYSCS_BACKUP_DATABASE
  • SYSCS_UTIL.SYSCS_RESTORE_DATABASE

And we've added these new system procedures:

  • SYSCS_UTIL.SYSCS_CANCEL_DAILY_BACKUP
  • SYSCS_UTIL.SYSCS_DELETE_BACKUP
  • SYSCS_UTIL.SYSCS_DELETE_OLD_BACKUPS
  • SYSCS_UTIL.SYSCS_SCHEDULE_DAILY_BACKUP

Finally, there are now several system tables that store information about backups:

  • SYSBACKUP System Table
  • SYSBACKUPITEMS System Table
  • SYSBACKUPJOBS System Table
1.50 Importing Data

There's now a how-to overview topic about Importing Data in this Guide. There are a number of updates and improvements:

  • The SYSCS_UTIL.SYSCS_IMPORT_DATA system procedure, which was available in earlier versions of Splice Machine, has been deprecated, and is replaced by the SYSCS_UTIL.IMPORT_DATA system procedure.
  • SYSCS_UTIL.IMPORT_DATA features new improvements, especially in error checking.
1.50 Limit clause Splice Machine now supports the LIMIT n clause in Select expressions, which you can use to limit the results of a query to a specific range of results.
1.50 Regular expression matches We have added the REGEXP_LIKE Operator, which is similar to the LIKE operator, except that it allows you to specify a full regular expression pattern match.
1.50 Running splice> from a file

You can invoke the splice> command line interpreter to run a set of SQL commands from a file and then exit, using the new -f argument.

For more information, see the Command Line (splice>) Reference
1.50 Savepoints

Transaction savepoints have been added. There's a brief description of savepoints in the Running Transactions In Splice Machine topic in the Developer's Guide.

There are also three new statements that are documented in the Transaction Commands section of the Developer's Guide

  • Savepoint Command
  • Release Savepoint Command
  • Rollback to Savepoint Command
1.50 Statistics

Statistics collection has been added to Splice Machine, for enhanced query optimization. The Using Statistics with Splice Machine topic in the Performance Tuning section introduces you to the functions and tables you can use to work with statistics for your database. And the SQL Reference Manual includes these reference topics:

  • SYSCS_UTIL.COLLECT_SCHEMA_STATISTICS built-in system procedure
  • SYSCS_UTIL.COLLECT_TABLE_STATISTICS built-in system procedure
  • SYSCS_UTIL.DISABLE_COLUMN_STATISTICS built-in system procedure
  • SYSCS_UTIL.DROP_SCHEMA_STATISTICS built-in system procedure
  • SYSCS_UTIL.DROP_TABLE_STATISTICS built-in system procedure
  • SYSCS_UTIL.ENABLE_COLUMN_STATISTICS built-in system procedure
  • SYSCOLUMNSTATISTICS System Table
  • SYSTABLESTATISTICS System Table
1.50 Temporary Tables

Temporary tables have been updated for compatibility with Business Intelligence tools, and there's a new how-to overview topic Using Temporary Database Tables in the Developer's Guide:

  • The CREATE TEMPORARY TABLEstatement has been added.
  • There are important updates to the DECLARE GLOBAL TEMPORARY TABLE statement.
1.50 Top clause Splice Machine now supports the TOP n clause in Select expressions, which you can use to limit the results of a query.
1.50 Triggers

Triggers are now operational. There's a new topic in the Developer's Guide: Using Database Triggers.

And there are updates for triggers in these SQL Reference Manual topics:

  • CREATE TRIGGER statement
  • DROP TRIGGER statement
  • UPDATE statement
  • WHERE clause
1.50 Window Functions

We've added four new window functions: 

  • FIRST_VALUE function
  • LAG function
  • LAST_VALUE function
  • LEAD function
