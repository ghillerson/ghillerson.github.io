The splice> Command Line Interface

This topic introduces the Splice Machine command line interface, which is also referred to as the splice> prompt; it contains a summary of the available commands.

The Splice Machine Command Line Reference is a complete reference manual for the splice> command line interpreter; it:

  • specifies the syntax rules of the command line interpreter
  • tells you how you can script splice> commands to run a series of operations like loading a number of files into your database.
  • contains a Commands Reference section with a reference page for each of the splice> commands.

Starting the splice> Command Line Interface

You start the splice> prompt by running the sqlshell.sh script, the location of which depends on your platform:

Hadoop platform Location of sqlshell.sh Notes
CDH 5.7.2, 5.8.0, 5.8.3

/opt/cloudera/parcels/CDH/bin/sqlshell.sh

You may need to create symbolic links to the JDK that was installed by Cloudera Manager so that the JDK is in the system path on each node, if it's not already there. Use commands like these:

sudo ln -s /usr/java/jdk1.7.0_67-cloudera /usr/java/default
sudo ln -s /usr/java/default/bin/java /usr/bin/java
HDP 2.4.2 and 2.5

~/sqlshell.sh

The sqlshell.sh script is found in the home directory of the cluster administrator user on each RegionServer node.
MapR 5.1.0 and 5.2.0

~/sqlshell.sh

The sqlshell.sh script is found in the home directory of the cluster administrator user on each RegionServer node.
Standalone version splicemachine/bin/sqlshell.sh

If you are running the standalone version of Splice Machine on a computer that goes into sleep mode, you will need to exit the command line, stop your database, and then restart it.

Running a file of SQL commands with the splice> Command Line Interface

You can invoke sqlshell.sh with the -f argument to start the command line interpreter, run a set of splice> commands stored in a file, and then exit. The following is an example of using this option:

$ ./sqlshell.sh -f /home/mydir/sql/test.sql

Commands Summary

The following table summarizes the commands you can enter in response to the splice> prompt. You can find a reference page for each command in the Commands Reference Click a command name in the table to view the command's reference page.

Command Description Usage
Analyze Collects statistics for a table or schema splice> analyze table myTable;
splice> analyze schema myschema;
Autocommit Turns the connection's auto-commit mode on or off. splice> autocommit off;
Commit Commits the currently active transaction and initiates a new transaction. splice> commit;
Connect Connect to a database via its URL. splice> connect 'jdbc:splice://xyz:1527/splicedb';
Describe Displays a description of a table or view. splice> describe myTable;
Disconnect Disconnects from a database. splice> disconnect SPLICE;
Elapsedtime Enables or disables display of elapsed time for command execution. splice> elapsedtime on;
Execute Executes an SQL prepared statement or SQL command string. splice> execute 'insert into myTable(id, val) values(?,?)' ;
Exit Causes the command line interface to exit. splice> exit;
Explain Displays the execution plan for an SQL statement. splice> explain select count(*) from si;
Export Exports query results to CSV files. splice> EXPORT('/my/export/dir', null, null, null, null, null) SELECT a,b,sqrt(c) FROM join t2 on t1.a=t2.a;
Help Displays a list of the available commands. splice> help;
MaximumDisplayWidth Sets the maximum displayed width for each column of results displayed by the command line interpreter. splice> maximumdisplaywidth 30;
Prepare Creates a prepared statement for use by other commands. splice> prepare seeMenu as 'SELECT * FROM menu';
Release Savepoint Releases a savepoint. splice> release savepoint gSavePt1;
Remove Removes a previously prepared statement. splice> remove seeMenu;
Rollback Rolls back the currently active transaction and initiates a new transaction. splice> rollback;
Rollback to Savepoint Rolls the current transaction back to the specified savepoint. splice> rollback to savepoint gSavePt1;
Run Runs commands from a file. splice> run myCmdFile;
Savepoint Creates a savepoint within the current transaction. splice> savepoint gSavePt1;
Set Connection Allows you to specify which connection is the current connection splice> set connection sample1;
Show Displays information about active connections and database objects. splice> show connections;
