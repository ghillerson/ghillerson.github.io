Documentation Summary

The Splice Machine Documentation is separated into these books:

Section Description
Getting Started This Guide provides an overview of Splice Machine and helps you to quickly get going with using your database.
Installation Guide This Guide provides step-by-step instructions for installing Splice Machine on various platforms.
Command Line (splice>) Reference The reference manual for our command line interface.
SQL Reference The complete reference for Splice Machine SQL.
Developers

Contains a number of sections for database developers:

  • Developer Fundamentals contains topics describing the fundamentals of developing your database with Splice Machine.
  • Connecting Your Database shows you how to connect to specific Business Intelligence tools, and how to use JDBC and ODBC with Splice Machine.
  • Functions and Stored Procedures contains a number of topics of interest to advanced database developers.
  • Tuning and Debugging contains topics that help you to optimize and debug your queries.
  • Tutorials contains an ever-expanding list of tutorials to help you zero in on the best ways to work with Splice Machine.
Administrators

The Administrator's Guide contains a number of sections for database administrators:

  • Configuring Splice Machine tells you how to configure database attributes such as logging, authentication, and user authorization.
  • Importing Data tells you how to import data into your Splice Machine database.
  • Performance Tuning describes Splice Machine's statistical collection and analysis tools that you can use to tune performance of your database.
  • Maintaining Splice Machine describes how to performance periodic maintenance tasks on your Splice Machine database.
  • Release Notes contains release and update notes for the current release, and for previous releases.
The Splice Machine Management Console Describes and shows you how to use the Splice Machine Management Console, which is a browser-based tool for monitoring and reviewing the performance of your queries.
Notes

Release notes, glossary, license information, and other noteworthy information.

Typographic Conventions Used in Our Books

We use the following style conventions in the Splice Machine books:

Style Usage
Code font

Fixed width font used for all code and any literal information that you might need to enter. For example:

A table named Players with primary key player_id
Application command

Anything that represents a specific application or is displayed by a specific application. For example:

hbase -> Actions -> Stop
hdfs1 -> Actions -> Stop
zookeeper1 -> Actions -> Stop
Example

Code and command examples. For example:

splice> CREATE TABLE NewPlayers
   AS SELECT * 
         FROM Players WITH NO DATA;
0 rows inserted/updated/deleted
Highlighted text
(Highlighted code font)

Text that is a placeholder, showing you that you need to substitute your own value for the highlighted portion. For example:

Maximum Client Connections = 0
Maximum Session Timeout    = 120000
Shell command

Text displayed by an operating system. For example:

sudo -su hdfs hadoop fs -rm -r /hbase

Different Views for Different Screen Sizes

Depending on the width of your screen or window (the display), the menu is displayed as a drop-down from the top of the display or as a pull-aside menu that automatically expands and collapses on the right side of the display. If you're using a computer with a screen wider than 1280 pixels, you can see this by reducing and enlarging the width of your browser window. We use three different views:

View name Window width range (in pixels)
Desktop >= 1280
Tablet 768 - 1279
Mobile <= 767
